## 🤔 Neden "Bana Kod Yaz" Demek Artık Eskide Kaldı?

Günümüzde herkes yapay zekaya kod yazdırıyor. Ancak ChatGPT'ye "Bana e-ticaret sitesi yap" dediğinizde çuvallıyor. Çünkü profesyonel projeler binlerce satırdan ve katı mimari kurallardan oluşuyor. Aldığınız kodu kopyalamak, dosya dosya yerine yerleştirmek ve hata ayıklamak aslında "yapay zeka hamallığı" yapmaktır.

*(Kendi Yorumun: Dev4All projesinden örnek verebilirsin. Mimari yapıyı kendin kurduğunu ama amelelik sayılabilecek işleri Agent'lara nasıl yıktığını anlat. "Ben mimarım, o işçi" metaforunu kullanabilirsin.)*

---

## 🤖 Agentic Coding (Otonom Kodlama) Nedir?

Agentic Coding, yapay zekayı bir kod üretici (jeneratör) yerine, sisteminizde gezinen, dosyaları okuyan, yeni dosyalar oluşturan ve işi bittiğinde gelip onayınızı isteyen bir **Kıdemli Yazılımcı (Senior Developer)** gibi kullanmaktır. 

Bizim yöntemimizde olay 3 adımda gerçekleşir:
1. **Böl ve Yönet:** Koca projeyi değil, sadece o anki işi (Örn: Sadece giriş sayfası) GitHub Issue olarak açarız.
2. **Kural Koy (Prompt):** Ajanın dışına çıkamayacağı mimari kuralları veritabanından (AGENTS.md) okumasını emrederiz.
3. **Arkanıza Yaslanın:** Agent'a görevi verip, onun arka planda dosyaları nasıl değiştirdiğini film izler gibi izleriz!

## 🗂 Sizin de Kullanabileceğiniz Şablonlarım (Copy-Paste Prompts)

Sadece fikri vermekle kalmıyor, benim kendi projelerimde kullandığım "Ajan Çalıştırma Talimatlarımı" sizinle de paylaşıyorum. Açtığınız GitHub Issue'larına bu taslakları yapıştırabilirsiniz:

### 🧩 Frontend (Arayüz) Şablonu
```markdown
Sen, bu projenin Frontend ekibindeki Senior React geliştiricisisin.
Amacımız, kurallarımızı bozmadan istenen bileşeni oluşturmaktır.

Görev (Task):
- Modern, koyu tema uyumlu bir [Buraya İstediğiniz Şeyi Yazın] tasarla.

Kurallar & Kısıtlamalar:
1. 'any' tipi kullanma.
2. Yazdıktan sonra build al ve bana Issue'yu kapatan bir PR onayı gönder.
```

### ⚙️ Backend Şablonu
```markdown
Sen, Senior bir Backend (.NET) geliştiricisin.
Clean Architecture ve SOLID prensiplerine harfiyen uy.

Görev:
- [Buraya İşlemi Yazın, Örn: Ürünleri çağıran API] yaz.

Kurallar:
1. Controller içine iş mantığı yazma, Application katmanını kullan.
2. İşlemi asenkron (async/await) yap.
```
